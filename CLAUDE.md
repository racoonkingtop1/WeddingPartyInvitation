# Инструкции для Claude в этом проекте

## Дизайнерские задачи — используй скиллы автоматически

Любая работа с UI/UX в этом репозитории (новая секция, правка Tailwind-классов,
цвет, типографика, спейсинг, компонент, лейаут) — это дизайнерская задача.
Для таких задач **до начала работы и перед тем как отчитаться о готовности**
всегда применяй проектные скиллы из `.claude/skills/`, без отдельного запроса
пользователя:

1. **`design-system-spec`** — сначала свериться с [`DESIGN.md`](./DESIGN.md):
   палитра, типографика, компонентные паттерны, анти-паттерны. Это источник
   истины, а не память из прошлых ответов.
2. **`design-system-generator`** — при выборе цвета/шрифта/паттерна для нового
   элемента опираться на подобранные в `DESIGN.md` токены, а не на дефолтные
   Tailwind-цвета или произвольные решения.
3. **`design-critique`** — после того как визуальный блок построен или
   отредактирован, прогнать чек-лист самопроверки из скилла, прежде чем
   называть задачу законченной.
4. **`caveman-mode`** — не авто-применяется; включается только если
   пользователь явно попросит короткий/грубый стиль ответов.

Если правки не затрагивают визуал (только логика, данные, конфиг деплоя) —
эти скиллы не нужны.

## Прочее

- Палитра, шрифты и компонентные паттерны зафиксированы в `DESIGN.md` —
  не вводить новые цвета/шрифты без обновления этого файла.
- `EVENT_CHAT_URL` (`src/data.ts`) и QR-код в `QRBlock.tsx` — намеренные
  плейсхолдеры, ждут реальных данных от пользователя.
